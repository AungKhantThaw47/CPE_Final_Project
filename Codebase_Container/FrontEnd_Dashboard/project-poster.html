Chulalongkorn University
Computer Engineering · Senior Project · 2026
Google Cloud Platform Gemini AI Gemma Embeddings Terraform · Neo4j
🚨
The Problem
Myanmar's humanitarian crisis generates daily Burmese-language reports on airstrikes, armed conflict, and displacement — but the data stays fragmented and unstructured.
🤖
Our Approach
An end-to-end automated pipeline that crawls, cleans, classifies, and annotates Burmese news using ML and generative AI, then plots confirmed events on an interactive map.
🎯
The Goal
Enable faster, data-driven humanitarian response by turning unprocessed text into structured, searchable, geolocated crisis event records — with zero manual triggering.
🔄 7-Stage Data Pipeline
🌐
① Crawl
Node.js scraper collects yesterday's articles from DVB Burmese news, triggered automatically every day at midnight via Cloud Scheduler.
Node.js Cloud Scheduler
🧹
② Clean
Strips HTML, ads, and duplicate content from raw articles, producing clean Burmese plain-text for the ML stage.
Python Cloud Run Job
🧠
③ Classify
Gemma-300M multilingual embeddings + scikit-learn pipeline predicts crisis vs. non-crisis for each article.
Gemma 300M sklearn · 16 GB
👤
④ Admin Review
Human-in-the-loop web service where analysts confirm or reject classifier predictions before annotation proceeds.
Crisis Admin Service Human-in-the-Loop
✨
⑤ Annotate
Gemini 3 Flash, guided by a 13-rule prompt, wraps each crisis event in <event> tags with precise boundaries.
Gemini 3 Flash Eventarc Trigger
📋
⑥ Extract
Gemini converts tagged events into structured JSON: location, date, type, fatalities, displacement, entities.
Structured JSON Gemini API
🗺️
⑦ Visualise
Leaflet.js map + Chart.js charts display confirmed events with filters, timeline, and event detail panel.
Leaflet.js Chart.js OpenStreetMap

🧠 ML Crisis Classifier
Gemma-300M multilingual embeddings capture Burmese text meaning, fed into a scikit-learn pipeline for binary crisis classification.
Input: Cleaned Burmese article text
Embedding: Mean-pooled Gemma 300M token vectors
Output: Crisis / Non-Crisis classification
Tracking: MLflow registry · GCS artifact store

✨ Gemini AI Annotation
A 13-rule structured prompt tells Gemini 3 Flash exactly how to segment articles into crisis events using date, location, and incident boundaries.
input → Burmese article text
prompt → 13-rule event-boundary rules
output → <event> tagged paragraphs
extract→ structured JSON fields
Events grouped by same date + location + incident

🔗 Smart Infrastructure
Content-hash rebuild detection — Terraform only rebuilds Docker images when source code actually changes.
Neo4j dependency graph — every job and service is linked to the GCS buckets it reads and writes, providing full data lineage.
GitHub Actions CI/CD — auto plan on PR, auto deploy on merge. 9 containers via Cloud Build.
