Background & Motivation

Myanmar faces an ongoing humanitarian crisis from armed conflict, airstrikes, and displacement. Burmese-language news reports remain scattered, unstructured, and inaccessible to international analysts.

No central aggregation across outlets
Plain text — not queryable or mappable
Burmese-only, limiting global reach
Manual analysis takes hours or days

Challenges & Design Decisions
1
Burmese NLP scarcity — No pre-tokenised datasets. Gemma-300M multilingual embeddings encode meaning directly from raw Unicode, bypassing tokenisation.
2
Myanmar geocoding gaps — OSM coverage is sparse. Solved with a 200+ hardcoded township coordinate table as primary lookup before any API call.
3
Event-driven coordination — Eventarc bucket events chain all 7 stages without polling. GCS object creation acts as the completion signal between jobs.
4
Multi-event articles — One article often covers 3–5 incidents. The 13-rule Gemini prompt segments by date and location into independent <event> spans.
G
ML Crisis Classifier
Gemma-300M multilingual embeddings feed a scikit-learn pipeline for binary crisis classification of Burmese articles.
Input: Cleaned Burmese article text
Embedding: Mean-pooled Gemma 300M token vectors
Output: crisis / non-crisis + confidence score
Tracking: MLflow registry on GCS

Gemini AI Annotation & Extraction
A 13-rule prompt tells Gemini 3 Flash how to segment articles by date, location, and incident into precise <event> spans.
input → Burmese article (plain text)
prompt → 13 event-boundary rules
output → <event> tagged paragraphs
extract→ { date, location, type, casualties, displaced, entities }
Auto-triggered by Eventarc on GCS upload
One JSON record per event, stored in GCS

Key Outcomes

3
